# CARVES Matrix Packaged Install Smoke

The packaged-install smoke is:

```powershell
pwsh ./scripts/matrix/matrix-packaged-install-smoke.ps1 -ArtifactRoot artifacts/matrix/packaged
```

It builds local NuGet packages and installs five dotnet tools into a temporary tool path:

| Package | Version | Command |
| --- | --- | --- |
| `CARVES.Guard.Cli` | `0.2.0-beta.1` | `carves-guard` |
| `CARVES.Handoff.Cli` | `0.1.0-alpha.1` | `carves-handoff` |
| `CARVES.Audit.Cli` | `0.1.0-alpha.1` | `carves-audit` |
| `CARVES.Shield.Cli` | `0.1.0-alpha.1` | `carves-shield` |
| `CARVES.Matrix.Cli` | `0.2.0-alpha.1` | `carves-matrix` |

After installation, it delegates to `carves-matrix e2e` in installed-tool mode and verifies the full Guard -> Handoff -> Audit -> Shield chain against a temporary external git repository.

The stable Shield input remains `shield-evidence.v0`, generated by `carves-audit evidence`.

## What Success Means

Success means local package installation works: local `.nupkg` files can be installed as dotnet tools and used from an external repository without loading private CARVES project truth.

Success does not mean:

- the packages have been pushed to NuGet.org;
- the result benchmarks or rates AI model safety;
- the tools provide operating-system sandboxing;
- the result is hosted verification;
- the result enters a public leaderboard;
- Shield output is public certification;
- any source, raw diff, prompt, model response, secret, credential, or customer data was uploaded.
